## 연습 1: ARM 템플릿을 사용하여 함수 앱 및 인지 서비스를 배포합니다.

#### 작업 1: Azure Portal 열기

1. 작업 표시줄에서 **Microsoft Edge** 아이콘을 클릭합니다.

1. 열린 브라우저 창에서 **Azure Portal**(<https://portal.azure.com>)로 이동합니다.

1. 메시지가 표시되면 이 랩에서 사용할 Azure 구독의 소유자 역할이 있는 사용자 계정으로 인증합니다.

#### 작업 2: Azure Resource Manager 템플릿을 사용하여 인지 서비스 배포

1. Azure Portal의 좌상단 모서리에서 **리소스 만들기**를 클릭합니다.

1. **새** 블레이드 최상단의 **마켓플레이스 검색** 텍스트 상자에서 **Template Deployment** 를 입력하고 **Enter** 키를 누릅니다.

1. **모두** 블레이드의 검색 결과에서 **템플릿 배포(사용자 지정 템플릿을 사용하여 배포)** 를 클릭합니다.

1. **템플릿 배포(사용자 지정 템플릿을 사용하여 배포)** 블레이드에서 **만들기** 단추를 클릭합니다.

1. **사용자 지정 배포** 블레이드에서 **편집기에서 사용자 고유의 템플릿을 빌드합니다** 링크를 클릭합니다.

1. **템플릿 편집** 블레이드에서 **파일 로드** 를 클릭합니다.

1. **업로드할 파일 선택** 대화 상자에서 **\\allfiles\\AZ-301T01\\Module_02\\LabFiles\\Starter\\** 폴더로 이동하여 **cognitive-template.json** 파일을 선택하고 **열기**를 클릭합니다. 이렇게 하면 다음 콘텐츠가 템플릿 편집기 창에 로드됩니다.

```json
    {
        "$schema": "http://schema.management.azure.com/schemas/2015-01-01/deploymentTemplate.json#",
        "contentVersion": "1.0.0.0",
        "variables": {
            "serviceName": "[concat('cgnt', uniqueString(resourceGroup().id))]"
        },
        "resources": [
            {
                "apiVersion": "2017-04-18",
                "type": "Microsoft.CognitiveServices/accounts",
                "name": "[variables('serviceName')]",
                "kind": "TextAnalytics",
                "location": "[resourceGroup().location]",
                "sku": {
                    "name": "S1"
                },
                "properties": {}
            }
        ],
        "outputs": {
            "cognitiveEndpointUrl": {
                "type": "string",
                "value": "[reference(variables('serviceName')).endpoint]"
            },
            "cognitiveEndpointKey": {
                "type": "string",
                "value": "[listKeys(variables('serviceName'), '2017-04-18').key1]"
            }
        }
    }
```

1. **저장** 단추를 클릭하여 템플릿을 유지합니다.

1. **사용자 지정 배포** 블레이드로 돌아가서 다음 작업을 수행합니다.

    - **구독** 드롭다운 목록 항목을 기본값으로 설정합니다.

    - **리소스 그룹** 섹션에서 **새로 만들기** 옵션을 선택한 다음 텍스트 상자에 **AADesignLab1001-RG** 를 입력합니다.

    - **위치** 드롭다운 목록에서는 이 랩에서 리소스를 배포할 Azure 지역을 선택합니다.

    - **사용 약관** 섹션에서 **위에 명시된 사용 약관에 동의함** 확인란을 선택합니다.

    - **구매** 단추를 클릭합니다.

1. 배포가 완료될 때까지 기다린 후 다음 작업 진행합니다. 

1. Azure Portal의 허브 메뉴에서 **리소스 그룹** 을 클릭합니다.

1. **리소스 그룹** 블레이드에서 **AADesignLab1001-RG** 를 클릭합니다.

1. **AADesignLab1001-RG** 블레이드에서 블레이드 상단에 있는 **배포** 헤더를 찾아 **배포** 레이블 하단을 클릭합니다. 그러면 성공적인 배포 수가 나타납니다.

1. 배포 블레이드에서 최근 배포의 이름을 클릭합니다.

1. **Microsoft.Template- 개요** 블레이드에서 **출력** 을 클릭합니다. 
1. **Microsoft.Template - 출력** 블레이드를 통해 **cognitiveEndpointUrl** 및 **cognitiveEndpointKey** 출력의 값을 식별합니다. 나중에 랩에서 이러한 값이 필요하므로 이러한 값을 기록합니다.

#### 작업 3: 함수 앱 배포

1. Azure Portal의 좌상단 모서리에서 **리소스 만들기** 를 클릭합니다.

1. **새** 블레이드의 최상단의 **마켓플레이스 검색** 텍스트 상자에서 **Function App**을 입력하고 **Enter** 키를 누릅니다.

1. **모든 항목** 블레이드의 검색 결과에서 **함수 앱** 을 클릭합니다.

1. **함수 앱** 블레이드에서 **만들기** 단추를 클릭합니다.

1. 다음 **함수 앱** 블레이드에서 다음 작업을 수행합니다.

    - **앱 이름** 텍스트 상자에 전역적으로 고유한 이름을 입력합니다.
    
    - **구독** 드롭다운 목록 항목 설정은 기본값으로 둡니다.

    - **리소스 그룹** 섹션에서 **기존 항목 사용** 옵션을 선택하고 드롭다운 목록에서 **AADesignLab1001-RG** 를 선택합니다.

    - **OS** 섹션에서 **Windows** 단추를 선택합니다.

    - **호스팅 계획** 드롭다운 목록에서 **소비 계획** 항목이 선택되어 있는지 확인합니다.

    
    - **위치** 드롭다운 목록에서는 이전 작업에서 인지 서비스 인스턴스를 배포한 Azure 지역을 선택합니다.
    
    - **런타임 스택** 드롭다운 목록에서 **NET Core** 항목이 선택되어 있는지 확인합니다.
    
    - **저장소** 섹션에서 **새로 만들기** 옵션을 선택하고 스토리지 계정 이름의 기본값을 수락합니다. 

    - **Application Insights** 섹션에서 옵션을 **비활성화** 로 설정합니다.

    - **만들기** 단추를 클릭합니다.

1. 함수 앱 프로비전이 완료될 때까지 기다린 후 다음 작업을 진행합니다. 

1. Azure Portal의 허브 메뉴에서 **리소스 그룹**을 클릭합니다.

1. **리소스 그룹** 블레이드에서 **AADesignLab1001-RG** 를 클릭합니다.

1. **AADesignLab1001-RG** 블레이드의 리소스 목록에서 새로 프로비전된 함수 앱을 클릭합니다.

1. 함수 앱 블레이드에서 블레이드 상단의 **플랫폼 기능** 탭을 클릭합니다.

1. **플랫폼 기능** 탭에서 **일반 설정** 섹션의 **구성** 링크를 클릭합니다.

1. **애플리케이션 설정** 탭에서 **애플리케이션 설정** 섹션을 찾습니다. **+ 새 애플리케이션 설정** 링크를 클릭하고 다음 작업을 수행합니다.

    - **이름** 텍스트 상자에 **EndpointUrl**을 입력합니다.

    - **값** 텍스트 상자에 앞에서 식별한 **cognitiveEndpointUrl** 값을 입력하고 그 뒤에 **text/analytics/v2.0** 을 입력합니다.

1. **응용 프로그램 설정** 섹션에서 **새 설정 추가** 링크를 다시 클릭하고 다음 작업을 수행합니다.

    - **이름 입력** 텍스트 상자에 **EndpointKey** 를 입력합니다.

    - **값 입력** 텍스트 상자에 앞에서 식별한 **cognitiveEndpointKey** 값을 입력합니다.

1. 함수 앱 블레이드로 다시 이동하여 **플랫폼 기능** 탭에서 **코드 배포** 섹션의 **배포 센터** 링크를 클릭합니다.


1. **플랫폼 기능** 탭의 **코드 배포** 섹션에서 **배포 센터** 링크를 클릭합니다.

1. 나타나는 **배포 센터** 블레이드에서 **외부**를 클릭한 다음 **계속**을 클릭합니다.

1. **App Service 빌드 서비스**를 클릭하고 **계속**을 클릭합니다. 

1. **코드** 섹션이 표시되면 다음 작업을 수행합니다.

    - **리포지토리** 텍스트 상자에서 **https://github.com/azure-labs/cognitive-services-function** 을 입력합니다.

    - **분기** 텍스트 상자에 **master** 를 입력합니다.
    
        > **참고**: 분기 필드는 대/소문자를 구분합니다.

    - **리포지토리 유형**의 값을 **Git**로 설정합니다.
    
    - **개인 리포지토리**의 값을 **아니요**로 설정합니다.

    - **계속** 단추를 클릭합니다.

1. **종료**를 클릭하고 배포가 완료될 때까지 기다린 후 다음 작업을 진행합니다. 

    > **참고**: **배포** 탭을 모니터링하면 첫 번째 배포가 완료되었는지 확인할 수 있습니다. 이 탭은 자동으로 업데이트됩니다.

    - **위치** 드롭다운 목록에서 이 랩의 이전 연습에서 선택한 Azure 지역을 선택합니다.

#### 작업 2: 논리 앱 단계 구성

1. Azure Portal의 허브 메뉴에서 **리소스 그룹**을 클릭합니다.

1. **리소스 그룹** 블레이드에서 **AADesignLab1001-RG** 를 클릭합니다.

1. **AADesignLab1001-RG** 블레이드에서 이전 작업에서 만든 논리 앱을 나타내는 항목을 클릭합니다.

1. **논리 앱 디자이너** 블레이드에서 아래로 스크롤하여 **템플릿** 섹션의 **비어 있는 논리 앱** 타일을 클릭합니다.

1. **논리 앱 디자이너** 블레이드에서 창 상단의 **코드 보기** 단추를 클릭합니다.

1. **논리 앱 디자이너** 블레이드에서 비어 있는 논리 앱 JSON 템플릿을 검토합니다.

```json
    {
        "definition": {
            "$schema": "https://schema.management.azure.com/providers/Microsoft.Logic/schemas/2016-06-01/workflowdefinition.json#",
            "actions": {},
            "contentVersion": "1.0.0.0",
            "outputs": {},
            "parameters": {},
            "triggers": {}
        }
    }
```

1. 기본 JSON 템플릿을 HTTP 트리거가 포함된 다음 템플릿으로 바꿉니다(**\\allfiles\\AZ-301T01\\Module_02\\LabFiles\\Starter\\logic-app.json**).

```json
    {
        "definition": {
            "$schema": "https://schema.management.azure.com/providers/Microsoft.Logic/schemas/2016-06-01/workflowdefinition.json#",
            "actions": {},
            "contentVersion": "1.0.0.0",
            "outputs": {},
            "parameters": {},
            "triggers": {
                "manual": {
                    "inputs": {
                        "method": "POST",
                        "schema": {
                            "properties": {
                                "text": {
                                    "type": "string"
                                }
                            },
                            "type": "object"
                        }
                    },
                    "kind": "Http",
                    "type": "Request"
                }
            }
        }
    }
```

1. **논리 앱 디자이너** 블레이드에서 **디자이너** 단추를 클릭합니다.

    > **참고**: 이 시점에서 디자이너에 단일 단계가 표시 됩니다. 워크플로를 시작하는 "트리거" 단계입니다.

1. 디자이너에서 **+ 새 단계** 단추를 클릭합니다.

1. **작업 선택** 섹션에서 다음 작업을 수행합니다.

    - 검색 텍스트 상자에서 **Azure Functions** 를 입력합니다.

    - 검색 결과에서 **Azure 함수 선택** 이라는 작업을 선택합니다.

    - 다음 검색 결과 집합에서는 이 랩의 이전 연습에서 만든 Azure 함수 인스턴스를 선택합니다.

    - 검색 결과의 마지막 집합에서는 작업에 사용할 **DetermineLanguage** 함수를 선택합니다.

1. **DetermineLanguage** 단계에서 다음 작업을 수행합니다.

    - **고급 옵션 표시** 링크를 클릭하여 모든 옵션을 표시합니다.

    - **요청 본문** 텍스트 상자에 **@triggerBody()** 를 입력합니다.

    - **메서드** 드롭다운 목록에서 **POST** 옵션을 선택합니다.

1. 디자이너에서 **+ 새 단계** 단추를 클릭합니다. **작업 추가** 단추를 클릭하여 작업 추가 대화 상자를 엽니다.

1. **작업 선택**섹션에서 다음 작업을 수행합니다.

    - 검색 텍스트 상자에서 **Azure Functions** 를 입력합니다.

    - 검색 결과에서 **Azure 함수 선택** 이라는 작업을 선택합니다.

    - 다음 검색 결과 집합에서는 이 랩의 이전 연습에서 만든 Azure 함수 인스턴스를 선택합니다.
